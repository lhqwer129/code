### `ASCII码`

计算机内部，所有的信息最终通过二进制值表示，一个字节(byte)包含8个二进制位(bit)。

`ASCII`码由美国制定，旨在统一规定英文字符与二进制位之间的关系。

`ASCII`码一共包含128个字符的编码，比如`SPACE`是32（00100000）。这128个符号只占用一个直接的后7位，最前面以为统一规定为0



### 非ASCII编码

ASCII码128位表示英语是足够了，但是用来表示其它语言，128个字符是远远不够的，因此急需一种编码方式能囊括世界上所有的符号，每一个符号对应一个独一无二的编码，因此就有了unicode编码，并再此基础上发展出了utf8，utf16等编码



#### Unicode

Unicode 是一个很大的集合，现在的规模可以容纳100多万个符号，具体可查http://www.unicode.org/

Unicode只是一个字符集，只规定了符号对应的二进制编码，但是并没有规定其如何存储。比如，汉字`严`的 Unicode 是十六进制数`4E25`，转换成二进制数足足有15位（`100111000100101`），也就是说，这个符号的表示至少需要2个字节。表示其他更大的符号，可能需要3个字节或者4个字节，甚至更多。

如上所述，unicode存在两个严重的问题

* 如何区分ASCII和Unicode码
* 不同字符存储时需要的字节数是不一样的，如果unicode统一规定一个字符存储时所需字节数，将造成极大的浪费。由于unicode没有规定字符的存储方式，因此出现来多种基于unicode的存储方式



#### UTF-8

utf-8是互联网上使用最广的一种unicode实现方式。

utf-8最大的特点是它是一种边长的编码方式，它可以使用1-4个字节来表示符号，根据不同的符号来变化字节长度。

utf-8编码规则如下:

* 对于单字节符号，字节的第一位为0，后面四位为Unicode吗，与ASCII码相同
* 对于n>1字节的符号，第一个字节的前n位都设为1，第n+1位设为0, 后面字节的前两位一律设为`10`。剩下的没有提及的二进制位，全部为这个符号的 Unicode 码。

```
Unicode符号范围     |        UTF-8编码方式
(十六进制)        |              （二进制）
----------------------+---------------------------------------------
0000 0000-0000 007F | 0xxxxxxx
0000 0080-0000 07FF | 110xxxxx 10xxxxxx
0000 0800-0000 FFFF | 1110xxxx 10xxxxxx 10xxxxxx
0001 0000-0010 FFFF | 11110xxx 10xxxxxx 10xxxxxx 10xxxxxx
```

跟据上表，解读 UTF-8 编码非常简单。如果一个字节的第一位是`0`，则这个字节单独就是一个字符；如果第一位是`1`，则连续有多少个`1`，就表示当前字符占用多少个字节。

下面，还是以汉字`严`为例，演示如何实现 UTF-8 编码。

`严`的 Unicode 是`4E25`（`100111000100101`），根据上表，可以发现`4E25`处在第三行的范围内（`0000 0800 - 0000 FFFF`），因此`严`的 UTF-8 编码需要三个字节，即格式是`1110xxxx 10xxxxxx 10xxxxxx`。然后，从`严`的最后一个二进制位开始，依次从后向前填入格式中的`x`，多出的位补`0`。这样就得到了，`严`的 UTF-8 编码是`11100100 10111000 10100101`，转换成十六进制就是`E4B8A5`

### java char and byte

char 类型是一个单一的 16 位 Unicode 字符, 根据Unicode编码的规定，一个英文字符占用一个字节，一个中文字符占用两个字节。char可以通过如下三种方式来赋值: 1.中文字符 2.16进制编码 3.十进制

```
public static void main(String[] args) {
    char c1 = '严';
    char c2='\u4E25';
    char c3 = 20005;
    System.out.println(c1);
    System.out.println(c2);
    System.out.println(c3);
}
```
